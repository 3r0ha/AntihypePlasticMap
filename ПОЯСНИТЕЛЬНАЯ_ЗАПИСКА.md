## 1. АННОТАЦИЯ

Проект «antihype · Карта пластика» — это комплексная система автоматической детекции скоплений морского пластика на основе мультиспектральных спутниковых данных Sentinel-2. Система разработана для экспедиции Фёдора Конюхова на плоту через Тихий океан, где капитану необходимо оперативно определять зоны максимальной концентрации пластикового мусора.

**Ключевые достижения:**
- Полностью автоматизированный конвейер «координаты → карта пластика» за 3–5 минут
- Мультииндексная детекция (FDI + FAI + PI + NDVI + NDWI) с адаптивным пороговым методом
- Лагранжева модель дрейфа пластика с ансамблевым прогнозом на 72 часа
- Оптимизация маршрута плота с учётом дрейфа мусорных пятен
- 6 пользовательских интерфейсов, включая ультралёгкий (<50 КБ) для спутникового интернета
- Полностью бесплатный стек (Sentinel-2, Microsoft Planetary Computer, open-source)

**Объём реализации:** ~6 600 строк Python (ядро) + ~1 600 строк Vue.js (фронтенд), 18 модулей, Docker-деплой.

## 2. ОПРЕДЕЛЕНИЕ ПРОБЛЕМЫ

### 2.1. Контекст

Великое тихоокеанское мусорное пятно (Great Pacific Garbage Patch, GPGP) — крупнейшее скопление морского пластика площадью ~1,6 млн км². По данным Lebreton et al. (2018), в GPGP содержится более 80 000 тонн пластика. Экспедиция Фёдора Конюхова на плоту ставит задачу мониторинга и документирования этих скоплений.

### 2.2. Проблемы, обозначенные в ТЗ

| Ограничение | Наше решение |
|-------------|--------------|
| Ограниченный спутниковый интернет на борту | Ультралёгкий интерфейс <50 КБ (lite_web), предрендеренная PNG-карта |

## 3. НАУЧНАЯ НОВИЗНА

### 3.1. Мультииндексная каскадная детекция

В отличие от существующих решений, применяющих единственный спектральный индекс, наша система использует **каскадную мультииндексную детекцию** — последовательность из 6 фильтрующих слоёв:

```
Слой 1: Водяная маска       → (SCL == 6) OR (NDWI > 0), исключить SCL {4,5}
Слой 2: Солнечный блик      → Фильтр по B08/B03 и интенсивности
Слой 3: FDI порог            → Адаптивный: max(Otsu, μ+3σ, 0.02, 0.005)
Слой 4: NDVI фильтр          → -0.1 < NDVI < 0.2 (исключение водорослей)
Слой 5: Интенсивность        → B08 < 0.3, B11 < 0.25 (исключение суши)
Слой 6: Морфологический      → Удаление кластеров < 5 пикселей
```

Каждый слой снижает количество ложноположительных срабатываний. Комбинация NDVI-фильтра с FDI позволяет отличить пластик от биологического мусора (водоросли рода Sargassum), что является известной проблемой в литературе (Biermann et al., 2020).

### 3.2. Адаптивное пороговое значение

Вместо фиксированного порога FDI мы применяем **адаптивный метод Оцу** (Otsu, 1979) на распределении положительных значений FDI, комбинируя его со статистическим порогом (μ+3σ) и минимальным абсолютным порогом (0.005). Это обеспечивает устойчивую детекцию в различных океанографических условиях — от прибрежных вод с высокой турбидностью до открытого океана с низким фоновым сигналом.

```python
threshold = max(
    otsu_threshold(positive_fdi),    # Оптимальная бимодальная граница
    fdi_mean + 3 * fdi_std,          # Статистический порог (3σ)
    0.02,                            # Минимальный порог ТЗ
    FDI_ABSOLUTE_FLOOR               # Абсолютный минимум (0.005)
)
```

### 3.3. Лагранжева ансамблевая модель дрейфа

Для прогнозирования перемещения обнаруженного пластика разработана **ансамблевая лагранжева модель** с интеграцией Рунге-Кутты 4-го порядка (RK4):

**Физическая модель скорости частицы:**
```
v_total = v_current + v_stokes + v_leeway

v_current — поверхностное океаническое течение (Open-Meteo Marine API / HYCOM)
v_stokes  — волновой дрейф Стокса ≈ 1.3% от скорости течения
v_leeway  — ветровой снос пластика ≈ 2.5% от скорости ветра на 10 м
```

Коэффициент leeway = 2.5% основан на экспериментальных данных Isobe et al. (2014) для макропластика с низким профилем.

**Ансамблевая конфигурация:**
- N = 30 частиц на каждый хотспот
- Случайные возмущения: ±5% скорости течений, ±2.5% ветра
- Временной шаг: dt = 0.5 ч (RK4)
- Горизонт прогноза: 24, 48, 72 часа
- Выход: медианная траектория + 1σ зона неопределённости

### 3.4. Маршрутизация с учётом дрейфа

Уникальная особенность — **drift-aware route optimization**. Плот Конюхова дрейфует под действием тех же течений, что и пластик, но имеет активную тягу (~2 узла ≈ 1 м/с). Алгоритм:

1. **Прогнозирование позиций:** Для каждого хотспота рассчитывается положение через время прибытия плота
2. **Жадный ближайший сосед:** Начальное приближение маршрута
3. **Локальная оптимизация 2-opt:** Устранение пересечений маршрута
4. **Итеративная коррекция:** Пересчёт дрейфа для каждого сегмента

Это обеспечивает реалистичный маршрут, где плот перехватывает мусорные пятна в их прогнозируемых будущих позициях, а не в текущих.

### 3.5. Темпоральный анализ аномалий

Для отделения **новых** пластиковых скоплений от постоянных прибрежных особенностей (осадочные шлейфы, рифы) реализован модуль темпорального анализа:

- 90-дневная историческая базовая линия (медианный FDI)
- Текущий FDI минус базовая линия = аномалия
- Положительная аномалия = новый/транзитный мусор

## 4. МЕТОДОЛОГИЯ И НАУЧНОЕ ОБОСНОВАНИЕ

### 4.1. Источник данных: Sentinel-2 L2A

**Sentinel-2** — мультиспектральный инструмент Европейского космического агентства (ESA):
- 13 спектральных каналов (443–2190 нм)
- Пространственное разрешение: 10–60 м
- Временное разрешение: 5 дней (два спутника: S2A + S2B)
- Уровень L2A: атмосферная коррекция (Sen2Cor), продукт BOA-отражение

Доступ реализован через **Microsoft Planetary Computer** — бесплатную облачную платформу с STAC API и Cloud-Optimized GeoTIFF (COG).

### 4.2. Спектральные индексы

#### FDI — Floating Debris Index (основной)

Разработан Biermann et al. (2020), Scientific Reports 10:5364. Основан на аномальном отражении плавающего пластика в ближнем инфракрасном (NIR) диапазоне:

```
FDI = ρ_NIR − [ρ_RE2 + (ρ_SWIR1 − ρ_RE2) × (λ_NIR − λ_RE2) / (λ_SWIR1 − λ_RE2)]
```

| Канал | Длина волны | Роль |
|-------|-------------|------|
| B6 (Red Edge 2) | 740 нм | Базовая линия (левый якорь) |
| B8A (NIR narrow) | 865 нм | Целевой канал (аномалия пластика) |
| B11 (SWIR 1) | 1610 нм | Базовая линия (правый якорь) |

**Физический принцип:** Пластик (полиэтилен, полипропилен) имеет повышенный коэффициент отражения в NIR по сравнению с чистой водой, создавая положительную аномалию FDI. Вода практически непрозрачна в NIR (поглощение), поэтому любой плавающий материал создаёт контраст.

#### FAI — Floating Algae Index (вспомогательный)

Hu (2009), JGR Oceans. Чувствителен как к пластику, так и к водорослям:
```
FAI = ρ_B08 − [ρ_B04 + (ρ_B11 − ρ_B04) × (λ_B08 − λ_B04) / (λ_B11 − λ_B04)]
```

#### PI — Plastic Index (вспомогательный)

Kikaki et al. (2020), Remote Sensing 12:2648:
```
PI = ρ_B08 / (ρ_B08 + ρ_B04)
```

#### NDVI — фильтр водорослей
```
NDVI = (ρ_B08 − ρ_B04) / (ρ_B08 + ρ_B04)
```
NDVI > 0.15 → биологический мусор (Sargassum), исключается из детекции пластика.

#### NDWI — водяная маска
```
NDWI = (ρ_B03 − ρ_B08) / (ρ_B03 + ρ_B08)
```
NDWI > 0 → водный пиксель (McFeeters, 1996). Исключает ложные срабатывания на суше.

### 4.3. Маскировка облачности

Используется Scene Classification Layer (SCL) Sentinel-2 L2A — пиксельная классификация атмосферных условий:

| Класс SCL | Описание | Действие |
|-----------|----------|----------|
| 0, 1 | Нет данных / Насыщение | Маскировать |
| 2 | Тёмные пиксели | Оставить (вода) |
| 3 | Тень от облаков | Маскировать |
| 4, 5 | Растительность / Почва | Маскировать (суша) |
| 6 | Вода | Оставить |
| 7 | Не классифицировано | Оставить |
| 8, 9 | Облака | Маскировать |
| 10 | Тонкая дымка | Маскировать |
| 11 | Снег/лёд | Маскировать |

После маскировки создаётся **медианный композит** по нескольким датам — это автоматически подавляет остаточную облачность и выбросы.

### 4.4. Кластеризация хотспотов

Алгоритм `scipy.ndimage.label()` выделяет связные компоненты (8-связность) на бинарной маске пластика. Для каждого кластера рассчитываются:
- Центроид (lat, lon) — взвешенный по FDI
- FDI_max — максимальное значение в кластере
- Площадь (км²) — количество пикселей × площадь пикселя
- Количество пикселей
- Confidence — нормированный FDI [0..1]

Результат: топ-10 хотспотов, отсортированных по FDI_max убыванию.

### 4.5. Модель океанических течений

**Трёхуровневый fallback источников данных:**

1. **Open-Meteo Marine API** (приоритетный) — бесплатный глобальный сервис:
   - Поверхностные океанические течения (u, v) на сетке 0.5°
   - Ветер на 10 м (для расчёта leeway)
   - Обновление: каждые 6 часов

2. **HYCOM via ERDDAP** (резервный) — глобальная модель циркуляции океана:
   - Компоненты (u, v) поверхностного течения
   - Пространственное разрешение: ~1/12°

3. **Синтетическая модель круговоротов** (аварийный fallback):
   - Аппроксимация субтропических круговоротов по полушариям
   - Постоянные поля (u, v)
   - Используется при недоступности внешних API

**docker-compose.yml** определяет 3 сервиса:
- **api** — FastAPI на uvicorn (обработка данных, REST API)
- **frontend** — Vue.js бандл на nginx (статика)
- **nginx** — Reverse proxy (маршрутизация `/` и `/api/`)

### 6.2. Автономное развёртывание (standalone)

Для работы на борту без сервера предусмотрены:
- **Streamlit** — полнофункциональный Web UI (порт 8501)
- **Lite GUI** — Bottle + Leaflet (порт 8090), совместимо с PyInstaller
- **Lite Web** — HTML <50 КБ (порт 8088), для спутникового интернета
- **CLI** — терминальный интерфейс с выводом в файлы

### 6.3. REST API

```
GET  /health               → Проверка доступности
GET  /presets               → Список пресетов (8 локаций)
POST /analyze               → Запуск анализа (async)
GET  /analyze/{job_id}      → Получение результата (polling)
POST /drift                 → Прогноз дрейфа
POST /route                 → Оптимизация маршрута
```

API поддерживает асинхронный режим: клиент отправляет запрос, получает `job_id`, затем опрашивает статус каждые 2 секунды. Это позволяет фронтенду отображать прогресс и не блокировать UI.

## 15. ПРИЛОЖЕНИЯ

### Приложение А. Формулы спектральных индексов

**FDI (Floating Debris Index):**
```
FDI = ρ_NIR − [ρ_RE2 + (ρ_SWIR1 − ρ_RE2) × (λ_NIR − λ_RE2) / (λ_SWIR1 − λ_RE2)]
```
Где: ρ_NIR = B8A (865 нм), ρ_RE2 = B6 (740 нм), ρ_SWIR1 = B11 (1610 нм)

**FAI (Floating Algae Index):**
```
FAI = ρ_B08 − [ρ_B04 + (ρ_B11 − ρ_B04) × (λ_B08 − λ_B04) / (λ_B11 − λ_B04)]
```

**PI (Plastic Index):**
```
PI = ρ_B08 / (ρ_B08 + ρ_B04)
```

**NDVI:**
```
NDVI = (ρ_B08 − ρ_B04) / (ρ_B08 + ρ_B04)
```

**NDWI:**
```
NDWI = (ρ_B03 − ρ_B08) / (ρ_B03 + ρ_B08)
```
